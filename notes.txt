1. SOA Design Principles (L R A A D C S S)
L – Loose Coupling (minimal dependencies, isolation, API-based communication)
R – Reusability (shared services, avoid duplication, modular design)
A – Autonomy (independent deployment, control own logic/data)
A – Abstraction (hide implementation, expose interface/API only)
D – Discoverability (service registry, metadata, documentation/APIs)
C – Composability (combine services into workflows, orchestration)
S – Statelessness (no session storage, each request independent)
S – Scalability (independent scaling, load distribution)


2. AWS CAF (B P G P S O)
B – Business (business alignment, value realization, investment planning)
P – People (skills, training, roles/responsibilities, culture change)
G – Governance (compliance, policies, risk management, control mechanisms)
P – Platform (infrastructure, applications, data, cloud architecture)
S – Security (identity, data protection, threat detection, compliance)
O – Operations (monitoring, incident response, change management)


3. AWS Well-Architected Pillars (O S R P C S)
O – Operational Excellence (automation, monitoring, continuous improvement)
S – Security (IAM, encryption, logging, threat protection)
R – Reliability (fault tolerance, backups, recovery, multi-AZ)
P – Performance Efficiency (scaling, resource optimization, right services)
C – Cost Optimization (pay-as-you-go, rightsizing, cost monitoring)
S – Sustainability (energy efficiency, resource usage, carbon reduction)


4. AWS Cloud Design Principles (S T A M M D C U)
S – Stop guessing capacity (Auto Scaling, metrics-based decisions)
T – Test in production (canary releases, A/B testing, feature flags)
A – Automate everything (IaC, CI/CD pipelines, scripts)
M – Make small, frequent changes (incremental deployments, easy rollback)
M – Monitor everything (logs, metrics, tracing, alerts)
D – Design for failure (redundancy, retries, failover, resilience)
C – Cost optimization (tracking spend, efficient resource use)
U – Use serverless when possible (Lambda, managed services, no infra mgmt)



AWS Fargate
👉 “Run containers without managing servers”
Serverless compute for containers (ECS/EKS). No EC2 management needed. You define CPU/memory and AWS runs and scales it.

AWS Cost Explorer
👉 “Track and analyze AWS costs”
Shows where your AWS money is going and how discounts are applied.

AWS GuardDuty
👉 “Threat detection service”
Analyzes logs and network activity to detect suspicious behavior or compromised resources.

AWS Glue Studio
👉 “Visual ETL builder”
Design, transform, and prepare data workflows with minimal code.

AWS Backup
👉 “Centralized backup management”
Automates backups and restores across AWS services.

AWS DataSync
👉 “Automated data transfer”
Moves data between on‑prem, AWS, or AWS storage services (fast and repeatable).

AWS Application Migration Service
👉 “Lift-and-shift servers”
Replicates and migrates on‑prem servers to EC2.

AWS Systems Manager
👉 “Manage servers centrally”
Control EC2 and on‑prem servers without SSH/RDP. Useful for hybrid environments.
